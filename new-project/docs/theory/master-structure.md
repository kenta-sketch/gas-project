## 0. プロジェクト概要

**ゴール** ── クワッドマインド理論をベースとしたAI内蔵HRプラットフォームを構築。採用→配置→継続マネジメント→チーム編成→将来は顧客ペルソナまで、1本の理論で貫通。

**現フェーズ** ── 理論材料収集 + ゴールデンウィーク中のデモ画面開発(友人への可視化参照点)

**直近期日** ── 2026年5月6日頃(GW終わり)までに採用フェーズv1のデモを友人に提示

**法人・ブランド構造**
- 法人格: イーセル(健太さん代表既存会社)の新事業ラインとして立ち上げ
- 外向ブランド: Quad Mind(独自ブランド・サイト・positioning)
- 二層構造で「事業の独立性」と「会社管理の効率性」を両立
- 将来: 独立採算で十分大きくなった時点で別法人にスピンアウト想定(選択肢を閉じない)

**役割分担**
- 友人: 理論側(IP保持、深掘り、ルール、整合性) ── ライセンサー
- 健太さん: ビジネス側(製品化、販売、組織運営) ── ライセンシー
- AI(本ファイル管理者): マスター構造図維持 / デモ仕様書 / LLMプロンプト草案

## 2. 4階層マップ(理論〜操作)

### 理論基盤層
- **Quad Mind Theory**(友人提供PDF) ── A動物的感情 / B機械的感情 / C動物的理性 / D機械的理性、黄金ルート A→B→C→D
- **Quad Vision Theory**(友人提供PDF) ── ①停滞 / ②危機管理 / ③慢心 / ④挑戦、軸=未来想定×感情統合度
- **愛の哲学**(友人提供PDF) ── 愛=循環の機能、自己への適用も成立
- **健太さんOS v3.0** ── 武器B=調達力(他者との合成)、デフォルト象限はクアッドビジョン④挑戦型

### 戦略層
- **BRAIN JUICEエッセイ10章**(友人作成) ── AI時代にDが代替される→A/B/C統合が人間の価値。構造「見た目→信用→実行→継続→信頼→愛」=A→B→C→Dと同型
- **健太さんボイス(時代論)** ── 同テーマの一次素材
- **★4倍の数字** ── 「同じ一人でも配置で4倍以上の差が出る」(健太さん仮説、感覚値)。経営層pitchの核心数字。実証データの裏取りは別途必要

### 診断層
- **3タイプ理論的分解**(友人作成):
  - 理詰め型: D高 / B中 / C低 / A低
  - 承認欲求型: B高 / D中 / A中 / C低
  - ワガママ型: A高 / C中 / B低 / D低
  - 各々に強み・リスク・マネジメントレバー・KPIあり
- **IQ/EQ分解**(友人作成):
  - IQ ≒ D×Cの統合
  - EQ ≒ A×Bの統合
  - 既存HR言語との接続装置(Quad Mindは未知概念、IQ/EQは通じる)
- **スコア粒度**(サンプルPDFから判明): A/B/C/D各25点満点 + 5感情反応(不安/悲しみ/怒り/喜び/幸福、各5点満点)

### 操作層
- **質問群Q1〜Q9**(友人作成) ── 意思決定 / 動機 / リスク耐性 / ポジティブ抽出
- **BRAIN JUICEスライド20枚**(友人作成)
- **brainjuice v3.pptx**(友人提供、uploads配置)
- **自己分析レポートサンプルPDF**(友人提供) ── 出力品質ベンチマーク
- **マネジメントガイドサンプルPDF**(友人提供) ── 出力品質ベンチマーク

## 3. 横断軸の対応関係

### A→B→C→Dの構造貫通
営業 / HR / 評価 / 関係構築 ── すべてが同じ流れで説明できる。「1本の理論で全業界貫通」の核心。

### IQ/EQ翻訳装置
- D+C統合=IQ(認知能力)、A+B統合=EQ(対人能力)
- Quad Mindは未知概念だが、IQ/EQは人事領域で通じる→ピッチ時の翻訳装置

### 二重出力構造(サンプルPDFから判明)
- **本人用レポート** ── 寄り添い・地図トーン、評価ではなく自己理解
- **管理者用ガイド** ── 操作的・根拠付き、各NG/OKに「なぜ機能するか」紐付け
- 「同じ事実を、誰のために翻訳するか」で出力切替 ── 商品としての差別化軸

## 4. 採用システム3軸ビジョン(v1の核心)

**入力(3軸)**
1. 診断 ── Quad Mindアンケート
2. 履歴書 ── 過去経験から人格信号を抽出
3. 面接 ── 1回目で素の反応観察、システムが2-3回目の質問を提案

**出力(採用段階)**
- 募集職種への適合度(pos / neg)
- クロスフィット提案(別職種への適性)
- ★**整合性チェック** ── 3軸の食い違いから「演じている可能性」検出 ← 本丸機能、他社が持っていない核心
- 面接深掘り質問の自動生成

**出力(入社後 = サンプルPDFと同水準)**
- 自己分析レポート(本人用)
- マネジメントガイド(管理職用)

## 5. ビジネスモデル構造

### 価格戦略
- **エンタープライズtier前提** ── 月100-200万 × 10社 を狙う(× 100社 × 月10万 ではない)
- 価格変動軸: 登録人数、管理形態、セキュリティ要件
- API原価は低く、利益率高い構造になる見立て

### 構造的優位
- **継続課金SaaS** ── 月額×従業員数×継続率の3軸スケール
- **高スイッチングコスト** ── 蓄積データ=企業独自の人材インテリジェンス。社員の歴史が消えるから乗り換え不可
- **理論的正当化** ── Quad Mindは「環境で変わる」と前提→定期診断が理論的に必要→継続課金が「理論的に要請されている」と顧客に説明可能

### v1から組み込むセキュリティ要件(エンタープライズ前提のため)
- SSO連携(SAML / OIDC)
- データ暗号化(保存時 + 通信時)
- 監査ログ
- ロールベースアクセス制御(本人 / 上長 / 人事 / 役員の出し分け)
- 国内データセンター
- SOC2 / ISO27001取得計画(導入1年目で必要になる場合あり)

デモ自体には不要だが、デモ仕様書に「本番ではこれを入れる」と明記し、Claude Code側で初期アーキテクチャを拡張性のある形で組む判断材料にする。

## 6. 理論IPの4層保護モデル

| 層 | 内容 | 顧客への開示 |
|---|---|---|
| L1 出力結果 | レポート・ダッシュボード | 完全開示 |
| L2 理論フレーム | 4軸の概念説明・思想 | マーケで部分開示 |
| L3 理論機構 | スコア計算式・プロンプト設計・判定ロジック | 完全保護(プロダクト内のみ) |
| L4 理論創出方法 | 理論を拡張する方法論 | 友人のみ保持 |

エンタープライズ顧客が触れるのはL1〜L2まで。L3はプロダクト内で動くがブラックボックス。L4は友人だけが持つ。これで「ツールを使い続ける構造的必要性」が生まれる。Quad Mind理論を語るマーケ資料を作る時も、この階層を意識して書く必要がある。

## 7. Partnership設計(確定版)

### 構造
- **Option C ハイブリッド型** ── 友人がIP保持、イーセル(またはその新事業ライン)が独占ライセンス取得
- 友人の役割: シニアアドバイザー / ライセンサー
- 法人新設なし(将来スピンアウトの選択肢は保持)

### 報酬モデル
- **売上連動ロイヤリティ%のみ** ── 純ロイヤリティモデル(友人の希望により最低保証はなし)
- 一時金(ライセンス契約締結時) ── 別途検討
- ストックオプション少量 ── 検討事項

### 友人の働き方
- ブラジル / 日本の行き来自由
- 理論メンテナンスと方向性レビュー
- アート活動と理論作者ブランディングは個人で展開、それが商品PRと相互強化される構造を意図

### 契約上の commitment(健太さんの覚悟を会社の義務に格上げ)
1. **不可逆ライセンス** ── 会社側から一方的解除不可、契約違反による解除のみ
2. **承継義務** ── 事業譲渡・会社売却・健太さん死亡時もライセンス契約が継承される条項
3. **死亡時の継承先指定** ── 友人が亡くなった場合の支払先(配偶者・子・指定者)を明記
4. **アンチ迂回** ── Quad Mind理論を別ブランドで展開等の迂回ができないよう対象IP定義を広めに取る

健太さんの「理論作者には事業が動き続ける限り報酬を払い続ける」という覚悟を、個人の善意ではなく**組織として永続する仕組み**に格上げする。10年来の親友であっても、関係性を守るために契約で実装する。

### Marketing 上の効果
「この会社は理論作者に永続的にロイヤリティを払い続ける」というポリシーは、外部に発信できる物語として機能する。理論作者へのリスペクトが商品品質保証になり、IP軽視会社との差別化要因になる。エンタープライズ顧客への信頼構築にも効く。

## 8. ギャップ・未決定事項

### 確定した4ギャップ(操作層)
1. **癌候補**(組織を腐らせるタイプ)の検出機構が無い
2. **責任感の3形態**(D型=規範 / B型=期待応答 / A型=結果)を質問が分離できていない
3. **統合状態**を検出する質問が無い(取りたい人材を直接判定できない)
4. **後付け項目**(記憶力/外向内向/攻め守り)の位置づけ未定義

### 未決定事項
- デモ範囲: A(採用1人のみ) / **B(採用+1年後時系列、推奨)** / C(採用+ダッシュボードWF)
- 友人へのフィードバック方針(先に返す or こちらで設計後に擦り合わせ)
- 事業の射程(Small業界特化 / Medium中堅 / Largeプラットフォーム)
- ロイヤリティ%の具体数値
- ライセンス契約一時金の額
- 既存AIコンサル事業との関係(別事業として並走 / 何らかの統合)
- プロダクト型 vs 受託型(現状はプロダクト型寄り)

### 個人領域の保留事項
- 健太さん自身の循環(健康領域)の停止状態 → 事業判断と並走か事前か未決定

### 決定済み(直近で確定)
- ✓ pricing tier: エンタープライズ前提(月100-200万×10社モデル)
- ✓ partnership: Option C ハイブリッド型(ライセンス契約)
- ✓ 友人との報酬: 売上連動ロイヤリティ%(最低保証なし、純ロイヤリティ)
- ✓ 契約 commitment: 不可逆/承継/継承先指定/アンチ迂回の4条項
- ✓ 法人形態: イーセルの新事業ラインとして立ち上げ(将来スピンアウト想定保持)
- ✓ ブランド: Quad Mind独自ブランド(法人格とは別軸の二層構造)
- ✓ 理論IP保護: 4層モデル(L1完全開示 / L2部分開示 / L3完全保護 / L4友人のみ)
- ✓ デモ開発環境: 健太さん側 = Claude Code、AI側 = マスター構造図/仕様書/プロンプト
