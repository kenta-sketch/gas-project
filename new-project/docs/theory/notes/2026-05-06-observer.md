# Observer ─ A/B/C/Dを統合するメタ機能

## Observerは「第5のエンジン」ではない

> Observerとは、自分の中で発火しているA/B/C/Dを認識し、その出力が行動を支配する前に介入する**上位観察機能**である。

A/B/C/Dは「判断」を行う機能。Observerは「判断を観察する」機能。

例: 怒りが出た瞬間に「いま自分は怒っている」「ここで言い返すと関係が壊れる」「一度、言葉を選ぼう」と認識できる。Aは消えていない。Aに支配されていないだけ。

## なぜObserverが必要か

A/B/C/Dだけでは「切り替え」を説明できない。成熟とは、エンジンを高めることではなく、**状況に応じて切り替えること**。Observerがなければ、人は自分の一番強いエンジンに支配される。

| 強いエンジン | 観察できないと |
|---|---|
| A | 感情のまま走る |
| B | 人の目で止まる |
| C | 勘を絶対視する |
| D | 考えすぎて止まる |

## Observerと単独運転

問題はA/B/C/Dの高さではなく、**特定エンジンが他を押し潰しObserverがそれに気づけないこと**(=単独運転)。

| 単独運転 | Observerが介入すれば |
|---|---|
| A | 「いまAが強い、出すと壊れる、Dで言葉に変える」 |
| B | 「いまBが強い、本当はAでやりたい、まず小さく出そう」 |
| C | 「いまCで判断、過去経験ベース、Dで根拠整理」 |
| D | 「いまDが過剰、80点で動いてCで現場修正」 |

## 心理学的接続

Observerは以下と接続:
- **メタ認知**: 自分の認知を認知する
- **自己制御**: 衝動・感情・欲求を目的に合わせて調整
- **認知的再評価**: 出来事の意味づけを変える(が、まず気づきが必要)
- **心理的柔軟性**: 感情・思考に支配されず、状況に応じて行動を選べる

## Observerと決断点

Observerは常に完全に働いているわけではない。多くの場合、人間は自動反応で動く。**Observerが起動しやすいのは決断点**:

- 言い返すか、黙るか
- 挑戦するか、やめるか
- 押すか、引くか
- 動くか、考え続けるか
- 本音を言うか、隠すか

→ Observer訓練とは、**決断点に気づく訓練**。

## 「気づいているのに止められない」

Observer介入力 vs エンジン出力 のバランスで決まる。
- Observer介入力 > エンジン出力 → 介入できる
- Observer介入力 < エンジン出力 → 気づいていても支配される

> 自己理解だけでは足りない。Observerには**介入力**が必要。

### 介入力を決める3要素

1. **身体状態**: 睡眠不足・疲労・慢性ストレスでObserver介入力は下がる
2. **蓄積量**: 未処理の感情が溜まっていると、次の刺激で発火する。1on1や面談は感情の蓄積量を減らす役割
3. **訓練量**: 自分の反応を言語化・振り返り・記録すると、A/B/C/Dの癖が見える。Observerは反復で育つ

## 個人Observerの成熟段階

| Lv | 状態 |
|---|---|
| 0 | 無自覚 - エンジン出力と自己認識が一体化 |
| 1 | 事後認識 - 行動した後に気づく |
| 2 | 途中認識 - 反応の途中で気づける、介入可能 |
| 3 | 事前認識 - 刺激の瞬間に方向が見える、環境設計可能 |
| 4 | 自動統合 - 切り替えが自然に行われる |

## 会社のObserver(組織Observer)

> 個人のObserverが**メタ認知**なら、会社のObserverは**仕組み化**。

優秀な管理者の勘や社長の感覚ではない。**組織が自分自身の偏りを検知するための仕組み**。誰が見ても異常が分かる、誰が見ても介入基準が分かる、誰が見ても履歴が残る、誰が変わっても回る。

### 会社版A/B/C/D

| エンジン | 個人 | 会社 |
|---|---|---|
| A | 感情・欲求・衝動 | 売上欲求・現場熱量・突破力 |
| B | 社会評価・羞恥・関係 | 社風・空気・評価制度・人間関係 |
| C | 経験圧縮・勘 | 現場知・暗黙知・トップ営業の型 |
| D | 論理・構造・制御 | KPI・マニュアル・研修・ルール・会議 |

### 単独運転の組織病理

| 単独運転 | 症状 |
|---|---|
| A | 数字だけで詰める。コンプラ崩壊。短期成果と引き換えに信頼喪失 |
| B | 沈黙の組織。誰も本音を言わない。ミスを隠す。突然退職 |
| C | 属人化。「見て覚えろ」。人が辞めるとノウハウが消える |
| D | 会議・資料・ルール・KPI増加。現場が動けない |

### 会社Observerの6工程

1. **観察** - 結果ではなく流れを見る(売上が落ちる前に行動量、その前に報連相、その前に面談、その前に共有...)
2. **記録** - 観察したものを残す。記録されないものは組織には存在しない
3. **判定** - A/B/C/Dで分解する。同じ「売れない」でも原因は違う
4. **介入** - 原因に合わせて対処する(根性論ではなく構造に合わせて変える)
5. **学習** - 介入結果を見る。効いたか、副作用はあったか
6. **標準化** - うまくいったものを研修・マニュアル・1on1項目・チェックリストにする

### 組織A/B/C/Dの問題と介入

| 問題 | 介入 |
|---|---|
| A不足 | 刺激と小さな成功体験 |
| A暴走 | ルールと責任設計 |
| B過剰 | 心理的安全性と個別面談 |
| B不足 | 関係コストの認識 |
| C不足 | ケース反復と即時フィードバック |
| C属人化 | 録音・録画・言語化・型化 |
| D不足 | 手順化とチェックリスト |
| D過剰 | 現場検証と削減 |

### Observerは監視ではない

会社Observerを間違えると監視になる。

- スタッフの弱点を探す ✗
- ミスを責める ✗
- 数字で詰める ✗
- 本音を吸い上げて評価に使う ✗

> 会社Observerは**人を縛るものではなく、人を壊さずに伸ばすための仕組み**。
> 評価の道具ではなく、環境調整の道具でなければならない。

特にBを扱う時は注意:
- A/Bを欠陥のように固定化して使うこと
- 恥を利用して行動修正すること
- 恐怖や社会的不安を刺激すること

これらは避けるべき。

### 1on1とObserver

1on1は雑談でも説教でもない。**A/B/C/Dの状態を観察する場**。

#### Aを見る質問
- 最近、やる気が上がった瞬間は?
- 強く嫌だと感じた瞬間は?
- 悔しかったこと / 楽しかった行動 / 避けたくなった仕事は?

#### Bを見る質問
- 最近、言いにくかったことは?
- 失敗を見られるのが怖かった場面は?
- 誰かの期待が重くなっていないか?
- 安心して相談できているか?

#### Cを見る質問
- 最近、現場で違和感を覚えたことは?
- うまくいった時、何を見て判断したか?
- 失注やミスの前に何かサインはあったか?

#### Dを見る質問
- 判断基準は明確か?
- 手順は分かっているか?
- 次に変える一点は何か?
- 再現するために必要な条件は何か?

### 数字をA/B/C/Dで読む

| 数字パターン | A/B/C/D解釈 |
|---|---|
| 案内数が低い | A不足の可能性 |
| 案内多いが成約低い | C不足 / D不足 |
| 成約高いがクレーム多い | A暴走 / B不足 |
| 数字は悪くないが離職多い | B過剰 / D過剰 |

> 数字は答えではない。数字はObserverの材料。

## 結論

> Observerとは、いま何が自分を動かしているのかを知る機能である。

- 個人のObserverが弱いと、人は支配エンジンに飲まれる
- 会社のObserverが弱いと、組織は支配構造に飲まれる

> Aで動く / Bでつながる / Cで読む / Dで整える / **Observerで、それらを見て選ぶ**

> 強い会社とは、売上が高い会社ではない。一時的に勢いがある会社でもない。本当に強い会社とは、**自分自身の偏りに気づき、修正し続けられる会社**である。Observerはそのための上位観察機能。
