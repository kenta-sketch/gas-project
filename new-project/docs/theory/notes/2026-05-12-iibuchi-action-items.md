## 2. 実証データから見えた事実(122人)

### 2.1 軸スコアの分布(各 0-25)
| 軸 | 平均 | SD | 最小 | 最大 |
|---|---|---|---|---|
| A | 17.0 | 4.2 | 8 | 25 |
| B | 16.0 | 4.2 | 5 | 25 |
| C | 17.5 | **3.4** | 10 | 25 |
| D | 17.2 | 4.4 | 8 | 25 |

→ **C軸だけSDが小さい**(全員 14〜21 に集まる)= **弁別力が低い**

### 2.2 軸間相関(理論上は直交が理想)
```
      A     B     C     D
A    1.00  0.18 -0.01 -0.20
B    0.18  1.00 -0.18  0.08
C   -0.01 -0.18  1.00  +0.37  ← C と D が強相関
D   -0.20  0.08 +0.37  1.00
```

**含意**:
- C-D が +0.37 で**強相関** → 直感型と論理型は一緒に動く(現代的な「賢さ」次元)
- A-D が -0.20 → 感情と理性の対立構造あり
- B-C が -0.18 → 承認依存と直感は逆方向
- **4軸独立の前提が崩れている**

### 2.3 5感情と軸の相関
```
         怒     悲     不安    喜     幸福
A_合計   0.32   0.45   0.33   0.18  -0.00
B_合計   0.17   0.36  +0.47  -0.06  -0.30  ← B高 = 不安+不幸(不健全)
C_合計  -0.21  -0.19  -0.39  +0.32  +0.47  ← C高 = 幸福+不安低(健全)
D_合計  -0.03  -0.11  -0.21   0.02   0.03  ← D は感情と独立
```

→ **C高=健全プロファイル、B高=不健全プロファイル** が実証的に存在

### 2.4 回答スタイルの分布(122人の傾向)
- Modest(穏当・2-4のみ): **49%**
- Discriminant(識別・1-5幅広く): 28%
- Extreme(極端・1か5ばかり): 18%
- Neutral(中立・3ばかり): 4%
- Acquiescence(同意・全体高め): 1%

→ 飯淵さんの懸念「2-4型と1-5型でブレる」は **実在を確認**(穏当49%、極端18%)

### 2.5 1〜5各値の選択率
```
1: 7.9%    2: 21.4%    3: 16.4%    4: 32.7%   5: 21.5%
                                       ↑ 圧倒的最多
全体平均: 3.39(中央3より +0.39 = やや同意傾向)
```

→ 「4」を最多に選ぶ = 中庸でややポジティブな自己像を出すバイアス

### 2.6 質問の弁別力(SD)
**強い質問**:
- 「昔のトラウマや経験が現在の感情反応に影響」(B) SD=1.42 ← 最強
- 「感情表現がオープン」(A) SD=1.33
- 「物事を仕組みや構造で考える」(D) SD=1.32

**弱い質問**:
- 「危険や違和感をすぐに察知し、直感で回避できる」(C) **SD=1.02** ← 最弱
- 「感情に蓋をする癖」(B) SD=1.12
- 「経験から学び、同じ失敗を繰り返さない」(C) SD=1.13

→ **C軸の質問は全体的に「自分はそうだ」と肯定されやすい**(C軸スコアの過大評価)

## 4. 飯淵さんに考えてほしい論点(5つ)

### Q1. 強制選択式 v3.0 の「中立(d)」問題、理論側から何が起きたか説明できるか?
- 設計者2人とも中立(d)を多用してスコアが薄くなった
- これは「冷静に質問を読む人ほど中立を選ぶ」構造問題?
- それとも質問文・選択肢の表現が中立に誘導しすぎている?
- 修正するなら:中立(d)を減らす / 別の意味(明確な「該当なし」)に変える / Likert との併用にする、どれ?

### Q2. C軸質問の弁別力不足、どう設計し直す?
- 「危険を直感で回避」SD=1.02 など、ほぼ全員が肯定する質問が多い
- 「自分は直感がある」と人は答えやすい → C軸スコアが過大評価される
- 一方で「サバイバル術がある」SD=1.32 のように識別力がある質問もある
- C軸の質問群を作り直す方向で良いか?具体的にどんな表現が「自分は持ってない」と答えやすくなるか?
- C-D 相関 0.37 を考えると、Cの質問が D的判断と混同されている可能性も

### Q3. 「4軸独立」前提を緩めるか?
- 実証データでは C-D=+0.37、A-D=-0.20、B-C=-0.18
- **4軸完全独立は理論上の理想で、実体は2つの対立軸 (A⇔D, B⇔C) で構造化されている**
- これを仕様書に反映して「4軸表現は説明力のため維持、内部判定では相関補正を入れる」と明文化するか?
- 12タイプ判定の「C+D=設計型」「D+C=分析型」の区別は C-D 相関が高いと意味が薄れるが、これは飯淵さんの理論意図と合うか?

### Q4. 健全度指標(C高=幸福、B高=不安・不幸)を本人向けに表示するか?
- 122人実証で明確な相関が出た:
  - C軸高 → 幸福+0.47、不安-0.39
  - B軸高 → 不安+0.47、不幸-0.30
- これは「組織毀損プロファイル」より先に出てくる、健全度の指標
- 本人向け画面に「あなたはC寄りなので幸福度が高い傾向」のような表示はアリか?
- それとも本人向けはタイプ判定のみに集中して、これは管理者向けのみが良いか?

### Q5. 将来の強制選択式 v3.0 の扱い
- 仕様書とDBは保存済み(`docs/theory/`)
- 将来データが集まったら復活させる選択肢はある
- でも「2系統並走(Likert と強制選択式の両方に答えてもらう)」だと回答負荷が倍
- 復活させるなら何が必要条件か?(例: Likert で X人以上の蓄積 / 質問の中立選択肢を再設計 / 等)

## 5. 飯淵さんに具体的にやってもらいたい3つ

優先順位順:

### 🎯 ① C軸質問の見直し(最優先)
- 現状の C軸 8問のうち、特に弁別力が低い質問を作り直す

### 🎯 ② v3.0 仕様書の中立(d)選択肢の再評価

- これを直すアイデアがあれば(中立の文言修正 / 中立を別変数化 / 中立を排除して3択にする 等)
- 飯淵さん自身が iA=0 になった事実をどう解釈するか

### 🎯 ③ 健全度指標と組織毀損プロファイルの位置関係を整理
- 現状の G5(組織毀損プロファイル)は B高型の派生(承認略奪)、A未統合(衝動暴走)、D高型(ルール暴力)
- 122人データでは B高 = 不安+不幸 が明確に出る
- 「組織毀損」より前に **健全度指標** を出すべきか?
- 例: 健全度高 → 通常表示 / 健全度低 → 注意フラグ + 詳細(組織毀損プロファイル) のような階層構造
