# 120人 Likert 実証分析 + v3.0 差し戻し決定

健太さん運用の既存 Google フォーム(Likert 5段階・v1.0形式)で **122人の生回答** が蓄積されている事実を 2026-05-12 に発見。
同日、内山健太・飯淵賢男 二人で実証データを確認し、**v3.0 強制選択式の本実装(コード)を差し戻し、v1.0 Likert ベースを継続採用する**ことに合意。
v3.0 仕様書および実装ノートは `docs/theory/` 配下にアーカイブ保存し、将来の理論的検証材料として残す。

---

## 決定の根拠

### 観察1: v3.0 強制選択式は設計者本人で機能していなかった

| 被験者 | 主軸 | A | B | C | D | iA | eA | Switch | 判定 |
|---|---|---|---|---|---|---|---|---|---|
| 内山健太 | (D) | 0 | 1.3 | 3.9 | 10.5 | 14.6 | 12.5 | 4.2 | A抑圧型 |
| 飯淵賢男 | C | 10.5 | 9.2 | 17.1 | 6.6 | 0 | 8.3 | 25 | 直感型 |

問題:
- 内山さん: 全軸スコアが極端に低い(中立 d を多く選んだ結果)
- 飯淵さん: 内的A=0(熱意・鼓舞を強みとする経歴と矛盾)、Switch=25(満点)なのに統合判定が「単独運転」(判定ロジックの中間ゾーン未定義バグ)

両ケースとも、診断対象者の実像と数値が一致しない。
原因は **「冷静に質問を読む人ほど中立(d)を選びがち」** という強制選択式 v3.0 の構造的欠陥と、判定ロジック側の閾値設計の粗さ。

### 観察2: 既存 Likert v1.0 は 122人の蓄積運用で機能していた

回答者属性:
- 性別: 男性82, 女性40
- 年齢: 20代後半58, 〜20代前半28, 30代前半21, ほか
- 職業: 一般職55, 自営26, 管理職23, 経営者12, ほか

軸合計スコアの分布(各 0-25):
| 軸 | 平均 | SD | 最小 | 最大 |
|---|---|---|---|---|
| A | 17.0 | 4.2 | 8 | 25 |
| B | 16.0 | 4.2 | 5 | 25 |
| C | 17.5 | 3.4 | 10 | 25 |
| D | 17.2 | 4.4 | 8 | 25 |

→ 全軸とも有意な分散があり、識別が機能している。
→ 内山さんの v3.0 結果(全軸一桁)は **120人母集団から大きく外れる特異値** で、Likert なら平均的なスコアが出るはず。

### 観察3: 飯淵さんの懸念(回答スタイル差)は実在するが、Likert + 変数化で吸収可能

回答スタイル分類(122人):
| スタイル | 該当人数 | 比率 |
|---|---|---|
| Modest(穏当型・2-4のみ) | 60 | **49%** |
| Discriminant(識別型・1-5幅広く) | 34 | 28% |
| Extreme(極端型・1か5ばかり) | 22 | 18% |
| Neutral(中立型・3ばかり) | 5 | 4% |
| Acquiescence(同意型・高得点寄り) | 1 | 1% |

全2440回答の分布:
- 1: 7.9% / 2: 21.4% / 3: 16.4% / **4: 32.7%** / 5: 21.5%
- 全体平均: 3.39(中央3より +0.39 = やや同意傾向)

→ 飯淵さんの懸念「2-4型と1-5型でブレる」は実在(穏当型49%、極端型18%)。
→ ただし、同じ Likert スコアでも回答スタイルが異なるため、**第2層変数として Response Style Profile を実装すれば加点方式のブレを補正できる**。

## 4軸の実証構造(理論的に直交ではない)

軸間相関(122人):
```
      A     B     C     D
A    1.00  0.18 -0.01 -0.20
B    0.18  1.00 -0.18  0.08
C   -0.01 -0.18  1.00 +0.37  ← C-D 強い正相関
D   -0.20  0.08 +0.37  1.00
```

**重要発見**:
- **C-D が 0.37** = 直感型と論理型は一緒に動く(「賢さ」次元)
- **A-D が -0.20** = 感情と理性は対立構造
- **B-C が -0.18** = 承認依存と直感は逆方向
- **A-B が 0.18** = 感情系同士で弱く連動

→ 仕様書の「4軸独立」前提は実証的には崩れている。
→ 実態は **「A⇔D(感情⇔理性)」「B⇔C(承認⇔直感)」の2つの対立軸** で見るほうが正確。
→ ただし4軸表現は説明力が高いので維持。内部判定で相関補正を入れる方向。

## 5感情と軸の相関 ─ 健全度指標が見える

```
         怒     悲     不安    喜     幸福
A_合計   0.32   0.45   0.33   0.18  -0.00
B_合計   0.17   0.36  +0.47  -0.06  -0.30   ← B高=不安+不幸(不健全)
C_合計  -0.21  -0.19  -0.39  +0.32  +0.47   ← C高=幸福+不安低(超健全)
D_合計  -0.03  -0.11  -0.21   0.02   0.03   ← Dは感情と独立
```

含意:
- **C高 = 健全プロファイル**(幸福+0.47、不安-0.39)
- **B高 = 不健全プロファイル**(不安+0.47、不幸-0.30)
- **D高 = 感情と独立**(理性は感情を遮断する)
- **A高 = 感情起伏全般と相関**(怒悲恐の全部+)

→ 仕様書の「組織毀損プロファイル」は B高との相関で実証的に説明できる。
→ C+D 高い人 = 「賢く幸福」プロファイルが実証的に存在。

## 質問の弁別力ランキング(全20問・SD順)

**強い質問(差がつく)**:
1. 「昔のトラウマや経験が現在の感情反応に影響」(B) **SD=1.42** ← 最強
2. 「感情表現がオープンで、顔や声に出やすい」(A) SD=1.33
3. 「物事を仕組みや構造で考える癖がある」(D) SD=1.32
4. 「自分なりの人生のサバイバル術を持っている」(C) SD=1.32
5. 「目的やゴールを設定してから逆算して行動する」(D) SD=1.31

**弱い質問(みんな同じに答える)**:
- 「危険や違和感をすぐに察知し、直感で回避できる」(C) **SD=1.02** ← 最弱
- 「感情が湧いてもすぐに無意識に蓋をする癖がある」(B) SD=1.12
- 「経験から学び、同じ失敗を繰り返さないタイプ」(C) SD=1.13

→ **C軸の質問群は全体的に弁別力が低い**(平均SD=1.18 vs A=1.24, B=1.25, D=1.25)
→ 「自分は直感がある」とほぼ全員が肯定するため、C軸スコアが過大評価されやすい。
→ C軸の質問を作り直すか、重みを軽減するか、回答時間でブースト/減点する設計が必要。

## 次の実装フェーズ(優先順位)

### Phase 1: Response Style 第2層変数の実装(優先度: 高)
- 各回答者の1-5分布を計算
- Modest / Discriminant / Extreme / Neutral / Acquiescence の自動分類
- 採点結果と並列で出力
- タイプ判定の補正材料として使う(例: Extreme型のスコアは正規化、Neutral型は弁別力不足フラグ)

### Phase 2: 回答時間ロギング(優先度: 中)
- クライアントで `Date.now()` 差分を記録、質問IDごとに集計
- カテゴリ別平均・SD、長考質問の特定
- C軸の弁別力不足を補う指標として活用

### Phase 3: C軸の質問見直し(優先度: 中)
- 弁別力が低い C 質問(SD<1.15)を友人と協議
- 「サバイバル術」のような相対的に強い質問は維持
- 「危険を直感で回避」のような全員肯定型は文言調整

### Phase 4: 軸間相関を考慮した補正(優先度: 低)
- C-D 0.37 を反映した「真の C 単独成分」「真の D 単独成分」を抽出
- 12タイプ判定の C+D / D+C 区別を再検討

### Phase 5: 健全度指標の表示(優先度: 中)
- 「あなたは C 寄り(健全プロファイル)」のような実証ベースの表示
- B高ユーザーへの管理者向け注意喚起(現状の G5 と統合)

## 設計上の方針(明文化)

1. **v1.0 Likert を主実装として継続する**
2. **v3.0 強制選択式は「理論的に正しい方向」として尊重しつつ、データが揃うまではアーカイブ**
3. **Likert の弱点(回答スタイルバイアス・社会的望ましさ)は第2層変数化で吸収する**
4. **122人の実証データを起点に、設計判断は必ずデータと付き合わせる**
5. **「設計者バイアス」を診断システムの欠陥として認識する**(内山・飯淵の数値は母集団から外れる事実)
6. **C軸の質問群と判定ロジックは飯淵さんと協議のうえ見直し**
