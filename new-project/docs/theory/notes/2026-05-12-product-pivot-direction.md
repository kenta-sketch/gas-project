# 別会社議論からのプロダクト方向性

2026-05-12 夜、内山さんが別会社の方と議論した結果出てきた5つの方向性。
当日 Claude と整理し、Phase 1-4 を実装着手、Phase 5 は将来検討で確定。

---

## 議論で出た5つの方向性

| # | 内容 | 規模 | Phase |
|---|---|---|---|
| 1 | クアッドマインド診断を抽出して他社採用システムと結合(API化・SaaS化) | 大 | 5(保留) |
| 2 | 会社の理念/方向性をベースに組み込んで適正を出す | 中 | 3 |
| 3 | いつでも自分で診断再度できるようにする(セルフ診断ポータル) | 中 | 4 |
| 4 | 診断だけでできるモード(スタンドアロン診断) | 小 | 1 |
| 5 | マネジメント部分でも診断できる(社員向け継続診断) | 中 | 2 |

## 本質的な方向性

「採用専用プロダクト」から「**診断エンジンの SaaS化**」へのピボット。
コア診断を独立させ、3つのユースケース(自社採用 / 個人セルフ / 他社組込)が同じエンジンを使う構造へ。

```
                                ↓ ピボット
┌──────────────────────────────────────────────────────────────┐
│ 【コア】QuadMind 診断エンジン                                │
│   - 75問 Likert + 第2層変数 + AI個別分析                     │
│   - 会社プロファイル(理念・方向性)を入力に追加 ← 差別化点  │
└──────────────────────────────────────────────────────────────┘
              ↓                  ↓                  ↓
┌──────────────────────┐ ┌────────────────┐ ┌──────────────────┐
│ 採用フロー(現プロダクト)│ │ セルフ診断ポータル │ │ 他社採用システム │
│ 応募→面接→判定        │ │ 自分でいつでも     │ │ API連携で組込    │
│ + マネジメント         │ │ 受け直し可能       │ │ (Phase 5: 保留) │
└──────────────────────┘ └────────────────┘ └──────────────────┘
```

## 実装計画

### Phase 1: 診断だけのモード(基盤)
- 新規ルート: `/diagnose`
- フロー: 最小プロフィール → 75問 → 結果ページ
- 応募者管理機能なしのスタンドアロン版
- 結果は localStorage に保存 + 共有用URL発行
- **他フェーズの土台になる**

### Phase 2: マネジメントから診断依頼
- 既存 `/admin/manage/[id]` に「診断を実施」ボタン追加
- 社員に URL 発行(`/diagnose?employeeId=xxx&scenario=再診断` のような形)
- 受診結果が社員レコードの `diagnoses[]` に追加される
- 既存の1年後比較機能を流用

### Phase 3: 会社プロファイル
- `/admin/settings` に「会社プロファイル」セクション追加
  - 理念(自由記述)
  - 求める人物像(自由記述 or 軸別の理想値)
  - 4軸の理想バランス(A/B/C/D の重視度 1-5)
- AI個別分析(`/api/personal-insight`)の入力に会社プロファイル追加
- 結果に「会社適合度」項目を追加

### Phase 4: セルフ診断ポータル
- 新規ルート: `/self` または `/me`
- 簡易認証(メール or ニックネーム + ローカル)
- 自分の診断履歴一覧
- 「再診断」ボタン
- 「前回からの変化」表示(既存 `/api/compare` を流用)

### Phase 5: API化(保留)
- 公開API + APIキー認証
- Webhook / iframe / 結果JSON
- ドキュメント整備・課金フロー
- 商品化が必要なフェーズなので将来

## 同時に対応する技術課題

### A. 既存AIレポートのキャッシュ修正
- 現状: 自己分析レポート / マネジメントガイド / 1年後比較は **画面を開くたびに再生成**
- 修正: 一度生成したら `Diagnosis.reports.{self,manager}` と `Employee.compareReport` にキャッシュ
- 「再生成」ボタンで明示的に上書き可能

### B. 診断内容の差し替え可能性
- ユーザー要望: 「診断の内容がちょっと変わるかもしれないから、後で変更できる状態にしてほしい」
- 対応: 質問とスコアリングは既に `src/lib/questions.ts` と `src/lib/scoring.ts` に集約済み
- 追加対応: `Diagnosis.questionSetVersion` を保存(将来質問が変わっても過去診断が破壊されない)
- 質問差し替え時は **新規バージョンを並走** させて段階移行可能にする
